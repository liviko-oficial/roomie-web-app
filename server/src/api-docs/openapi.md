# Documentation


| **Método** | **Endpoint**                | **Descripción**                                                           | **Body Params**                                                                                                                                                                                                                                                                                                                                                   | **Query Params**                                                                                  | **Middleware / Restrictions**                                    | **Respuestas**                                                                                                                                                                                                                                                  |
|------------|-----------------------------|---------------------------------------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------|------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **POST**   | `/register`                 | Registro de usuario (email + password).                                   | - `email`: email válido; debe pertenecer a un dominio permitido<br>- `password`: string de 8–32 caracteres; debe incluir mayúscula, minúscula, número y carácter especial                                                                                                                                                                                         | - *(sin params)*                                                                                  | —                                                                | `200`: OK<br>`400`: (1) No se proporcionó contraseña, (2) Contraseña con formato incorrecto, (3) No se proporcionó cookie<br>`401`: El token no está verificado para el email proporcionado<br>`500`: Error interno del servidor                                |
| **GET**    | `/register/verify`          | Verifica token y registra usuario parcial.                                | - *(sin params)*                                                                                                                                                                                                                                                                                                                                                  | - `email`: email válido; debe pertenecer a un dominio permitido<br>- `token`: string; UUID válido | —                                                                | `200`: Verificación exitosa; usuario parcial creado (continúa flujo)<br>`400`: (1) Token o email inválidos, (2) Esquema de usuario parcial inválido                                                                                                             |
| **POST**   | `/login`                    | Login de usuario.                                                         | - `email`: email válido; debe pertenecer a un dominio permitido<br>- `password`: string de 8–32 caracteres; debe incluir mayúscula, minúscula, número y carácter especial                                                                                                                                                                                         | - *(sin params)*                                                                                  | - Requiere autenticación                                         | `200`: Login exitoso; redirección o confirmación<br>`400`: (1) Datos de envío inválidos, (2) Contraseña incorrecta<br>`401`: Cookie de sesión faltante<br>`404`: Usuario no encontrado                                                                          |
| **GET**    | `/login/upgrade-partial`    | Convierte usuario parcial en usuario completo.                            | - *(sin params)*                                                                                                                                                                                                                                                                                                                                                  | - *(sin params)*                                                                                  | - Requiere autenticación                                         | `200`: (1) Permisos no coinciden con el endpoint, (2) Upgrade exitoso (continúa flujo)<br>`400`: Error en el esquema de datos<br>`401`: Cookie de sesión faltante<br>`500`: (1) No se encontró usuario parcial para actualizar, (2) Error inesperado al guardar |
| **GET**    | `/user`                     | Obtiene información de usuario.                                           | - *(sin params)*                                                                                                                                                                                                                                                                                                                                                  | - *(sin params)*                                                                                  | - Requiere autenticación                                         | `200`: Información de usuario retornada (parcial o completa)<br>`401`: Cookie de sesión faltante<br>`500`: (1) Error al obtener usuario de la base de datos, (2) Error inesperado                                                                               |
| **DELETE** | `/user`                     | Elimina al usuario autenticado.                                           | - *(sin params)*                                                                                                                                                                                                                                                                                                                                                  | - *(sin params)*                                                                                  | - Requiere autenticación                                         | `200`: Usuario eliminado; sesión cerrada<br>`401`: Cookie de sesión faltante<br>`500`: (1) Error al eliminar usuario, (2) Error inesperado                                                                                                                      |
| **PUT**    | `/user`                     | Cambia propiedades generales del usuario.                                 | - `body`: objeto; propiedades válidas según `RealUser.partial()`                                                                                                                                                                                                                                                                                                  | - *(sin params)*                                                                                  | - Requiere autenticación                                         | `200`: Usuario actualizado<br>`400`: (1) Body ausente, (2) Error de validación de cambios<br>`401`: Cookie de sesión faltante<br>`500`: Error al guardar en base de datos                                                                                       |
| **PUT**    | `/user/preferences`         | Actualiza preferencias del usuario.                                       | - `preferences`: objeto; propiedades válidas según `UserResponse.partial().shape.preferences`                                                                                                                                                                                                                                                                     | - *(sin params)*                                                                                  | - Requiere autenticación                                         | `200`: Preferencias actualizadas<br>`400`: (1) No hay preferencias para cambiar, (2) Error de validación de preferencias<br>`401`: Cookie de sesión faltante<br>`500`: Error al guardar en base de datos                                                        |
| **POST**   | `/user/register/form`       | Carga formulario de registro extendido.                                   | - `hobbies`: lista de strings; mínimo 1 elemento<br>- `deal_breakers`: lista de strings; opcional<br>- `current_quarter`: número; valor entre 1 y 12<br>- `birth_date`: fecha válida<br>- `id`: número; identificador válido<br>- `gender`: string; valor permitido<br>- `has_pets`: boolean<br>- `smoker`: boolean<br>- `cellphone`: string; mínimo 8 caracteres | - *(sin params)*                                                                                  | - Requiere autenticación                                         | `200`: Formulario de registro guardado<br>`400`: Error en datos del formulario<br>`401`: Cookie de sesión faltante                                                                                                                                              |
| **GET**    | `/password-recovery`        | Envía email para recuperar contraseña.                                    | - *(sin params)*                                                                                                                                                                                                                                                                                                                                                  | - `email`: email válido; debe pertenecer a un dominio permitido                                   | —                                                                | `200`: (1) Email de recuperación enviado, (2) Usuario no encontrado (respuesta neutral)<br>`400`: Falta email<br>`500`: Error inesperado al enviar correo                                                                                                       |
| **GET**    | `/password-recovery/verify` | Verifica token para habilitar cambio de password.                         | - *(sin params)*                                                                                                                                                                                                                                                                                                                                                  | - `email`: email válido; debe pertenecer a un dominio permitido<br>- `token`: string; UUID válido | - Requiere token valido                                          | `200`: (1) Usuario no encontrado (respuesta neutral), (2) Verificación exitosa; continúa flujo<br>`400`: (1) Falta email o token, (2) Error de validación del esquema<br>`401`: Error de verificación                                                           |
| **PUT**    | `/password-recovery`        | Cambia la contraseña.                                                     | - `password`: string de 8–32 caracteres; debe incluir mayúscula, minúscula, número y carácter especial                                                                                                                                                                                                                                                            | - *(sin params)*                                                                                  | - Requiere password recovery cookie (dada en `/password/verify`) | `200`: Contraseña cambiada con éxito<br>`400`: (1) No se proporcionó contraseña, (2) Formato de contraseña incorrecto, (3) Falta cookie de recuperación                                                                                                         |
| **GET**    | `/`                         | Muestra un mensaje HTML indicando el acceso a la API.                     | - *(sin params)*                                                                                                                                                                                                                                                                                                                                                  | - *(sin params)*                                                                                  | —                                                                | `200`: Página informativa de acceso a la API                                                                                                                                                                                                                    |
| **GET**    | `/health-check`             | Verifica el estado del servidor; responde con “Server is running OK”.     | - *(sin params)*                                                                                                                                                                                                                                                                                                                                                  | - *(sin params)*                                                                                  | - Requiere autenticación                                         | `200`: Servidor en funcionamiento correcto<br>`401`: Cookie de sesión faltante                                                                                                                                                                                  |
| **ALL**    | `*`                         | Manejador 404; responde con HTML indicando que la ruta no fue encontrada. | - *(sin params)*                                                                                                                                                                                                                                                                                                                                                  | - *(sin params)*                                                                                  | —                                                                | `404`: Ruta no encontrada                                                                                                                                                                                                                                       |
